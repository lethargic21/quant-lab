# dart-event-study

DART 전자공시에서 이벤트(자사주매입 / 유상증자 / 실적공시)를 감지해 구조화 정보를
추출하고, 이벤트 스터디와 백테스트로 검증하는 엔드투엔드 파이프라인.
유니버스는 KOSPI200, 기간은 2019–2024. 상세 설계는 [PLAN.md](./PLAN.md).

> 🚧 WIP — 방법론·결과·한계는 파이프라인 완성 후(Phase 5) 채워진다.

## 방법론 (요약)

- 이벤트는 여론 감성이 아니라 **공시의 구조화 정보 + 방향 룰**로 처리한다.
- Look-ahead 방지: 시그널은 공시 접수시각 이후만 사용. 장중(≤15:30) 접수 → 당일 종가,
  장 마감 후 → 익영업일 체결 가능으로 보수적 가정 (유닛테스트로 검증).
- 수정주가 사용, 거래비용(거래세+슬리피지) 반영, 테스트한 윈도우/파라미터 전부 리포트.

## 재현 방법

## 한계

- 유니버스는 실제 KOSPI200이 아니라 **KOSPI 시총 상위 200 보통주의 현재 스냅샷**
  (KRX 지수구성종목 API가 봇 차단이라 조회 불가) — 과거 편입/편출 미반영 (survivorship bias).
- 가격은 FinanceDataReader 네이버 소스의 수정주가. 일별 시가총액은 무료 소스가 없어
  시총 대비 비율 대신 공시 내 주식수 기반 비율(취득주식수/발행주식총수 등)을 사용.
- 거래정지: 거래량 0인 날로 추정해 플래그. 상폐: 가격 데이터가 기간 끝보다
  20거래일 이상 일찍 끊기면 추정 플래그 — 정밀한 상폐 이력 DB는 사용하지 않음.
- 공시 접수 '시각'은 OpenDART 목록 API에 없음(일자만). 시각 미확보 시 전 건
  익영업일 체결의 보수적 가정 적용.
- 실적 서프라이즈는 무료 컨센서스 부재 시 YoY/QoQ 성장률 대용치 사용.

## Roadmap (범위 밖 — 코어 완성 후 검토)

- 이벤트에 대한 시장 반응 감성 레이어: 같은 자사주매입 공시라도 뉴스/커뮤니티 감성이
  뜨거운 종목의 이후 수익률이 다른가 (과잉반응→되돌림 vs 모멘텀 지속).
  소스 후보: 뉴스 기사 → 뉴스 댓글 → 종목토론방. 이번 구현에는 포함하지 않는다.
