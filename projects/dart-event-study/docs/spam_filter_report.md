## 1. 규칙 베이스라인 커버리지 (파라미터: dup≥3, burst=60분 내 2회, 정규화 최소 4자)

| 규칙 | 발화 | 비율 |
|---|---|---|
| flag_dup (정규화 후 동일 제목 코퍼스 3회+) | 2,694 | 21.0% |
| flag_burst (60분 내 동일 제목 2회+) | 2,568 | 20.0% |
| flag_marker (●■★ 등 홍보 기호) | 1,727 | 13.5% |
| flag_link (외부 URL) | 61 | 0.5% |
| flag_lead (리딩·무료체험·수익보장 문구) | 39 | 0.3% |
| flag_contact (카톡·오픈채팅·텔레그램) | 6 | 0.0% |
| **spam_rule (OR)** | **4,233** | **33.0%** |

- **게시판의 1/3이 규칙에 걸린다.** 종목별로도 31.0~35.9%로 균일 — 스팸이 특정 종목 문제가 아니라
  게시판 전반의 구조적 현상. attention 지표를 필터 없이 쓰면 안 된다는 사용자 우려가 실측으로 확인됨.
- 지배 규칙은 **dup/burst(도배)**와 **marker**. lead/contact 문구 규칙은 거의 발화 안 함 —
  리딩 영업이 "문구"가 아니라 **도배+기호**로 이뤄지기 때문.

## 2. 눈검사 관찰 (spam_rule=True 무작위 10건)

10건 전원이 사람 눈으로도 명백한 스팸이었고, 패턴이 뚜렷하다:
- **타 종목 리딩이 주류**: 삼성전자 게시판에 리센스메디컬·푸른로보틱스·케이씨에스·액스비스 등
  소형주 "폭등" 홍보 도배 — 유동성 높은 게시판을 광고판으로 쓰는 구조.
- 유튜브 링크 낚시, "오늘 공개 → 폭등 TOP 종목" 류 전형적 리딩 문구.
- 시사점: **"해당 종목 무관(타 종목 언급)" 자체가 강한 스팸 신호**인데 현재 규칙엔 없음 —
  dup/burst가 간접적으로 잡고 있으나, 도배 안 하는 단발 타종목 홍보는 놓칠 것(recall 저하 예상 지점).
  → 사람 라벨 후 오분류 분석에서 검증 예정.

## 3. 예상 약점 (사람 라벨로 검증할 가설)

1. **recall 저하 후보**: 기호·링크 없이 정중한 문장으로 쓴 단발 리딩글, 단발 타종목 홍보.
2. **precision 저하 후보**: 개인 유저의 습관성 반복글(같은 한탄을 매일 게시)이 dup에 걸림 —
   도배와 팬심의 경계. 라벨링 가이드 규칙 2로 판정 일관성 확보.
3. marker 단독 발화(●만 있고 내용 정상)의 오탐 여부.

## 4. 다음 단계

2. 라벨 수신 후: 규칙 베이스라인의 진짜 precision/recall/F1 (300건 기준) 산출.
3. KcELECTRA-base 파인튜닝 — 사람 라벨 층화 분할(train/holdout), 규칙 약라벨 병용 검토,
   **홀드아웃(사람 라벨)에서만 평가**. 규칙 vs 모델 비교 + 오분류 정성 분석으로 본 리포트 갱신.

## 5. 민감도 준비 (5단계 — 구현 완료)

- `SpamRuleParams(enabled=False)`로 필터 전체 off — attention 지표 전/후 비교 가능.
- 임계값(dup_min, burst_window_min, burst_min, min_norm_len) 전부 파라미터.
- 산출 parquet(`data/processed/{code}_labeled.parquet`)에 규칙별 플래그가 개별 컬럼으로 남아
  어떤 규칙 조합이든 사후 재구성 가능.
