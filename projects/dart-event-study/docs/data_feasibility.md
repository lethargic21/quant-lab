# 종목토론방 데이터 수집 가능성 정찰 리포트

> 2026-07-13 · 정찰 목적: 개인투자자 게시판 심리 → 주가 단기 선행관계 연구의 데이터 성립 여부.
> **크롤러 미구현 — 접근 가능성·타임스탬프 해상도만 확인.** 정직한 UA(연구 식별자), 요청 간격 2.5초+,
> 2종목·5페이지로 최소 정찰. 차단 우회·UA 위장·로그인 접근 일체 없음.

## 0. 판정 (한 줄)

- **네이버 금융 종목토론방 = 불가능** (robots.txt가 일반 크롤러에게 전체 `Disallow: /` — 게시판 미접근, 요청 0건).
- **팍스넷 종목토론실 = 조건부 가능** — 타임스탬프 **초 단위**로 연구 설계 성립. 단, **게시글 밀도가 극단적으로
  편중**(인기주만 유의미, 중·소형주 다수는 사실상 정지)되어 전 종목 횡단 심리 측정은 제약.

**연구 설계 관점 결론**: 식별 전략(장후~익일 개장 전 작성 글은 역인과 차단)의 **전제인 시각 해상도는 팍스넷에서
충족**된다(초 단위 + KST). 따라서 연구는 성립 가능하나, **표본이 소수 유동성 종목에 국한**된다.

---

## 1. [최우선] 타임스탬프 해상도

| 소스 | 목록 페이지 해상도 | 근거 |
|---|---|---|
| 네이버 금융 | **확인 안 함** (robots.txt 차단으로 미접근) | — |
| 팍스넷 | **초 단위 (HH:MM:SS + KST 타임존)** | 목록 HTML의 `data-date-format` 속성 |

팍스넷 게시글 목록의 각 행 날짜 칸에 다음이 **정적 HTML로** 들어있음:

```html
<div class="date"><span class="data-date-format"
      data-date-format="Mon Jul 13 12:47:09 KST 2026"></span></div>
```

- 화면 표시는 JS가 포맷하지만, **원본 값(초 단위 + KST)은 정적 HTML 속성에 그대로** 존재 → JS 렌더링 없이 취득 가능.
- 목록 페이지에서 이미 초 단위 → **상세 페이지를 따로 열 필요 없이** 장중(09:00–15:30)/장후 구분 가능.
- **연구 설계에 미치는 영향**: 식별 전략이 요구하는 "장 마감 후 ~ 익일 개장 전" 시간대 구분이 **초 단위로 정밀하게
  가능**. 날짜 단위였다면 설계를 바꿔야 했으나 그럴 필요 없음. → 이 항목은 **긍정 판정**.

## 2. 접근 가능성

### 네이버 금융 (finance.naver.com) — robots.txt 차단

```
User-agent: *
Disallow: /                                  ← 일반 크롤러: 전체 금지
User-agent: yeti                             ← 아래 Allow는 네이버 자체봇(Yeti) 전용
Disallow: /
Allow: /item/board.naver?code=*
Disallow: /item/board.naver?code=*&page=*    ← Yeti조차 페이지네이션은 금지
```

- 종목토론방 경로 `/item/board.naver`의 `Allow`는 **`User-agent: yeti` 그룹에만** 속함. 일반 크롤러(`*`)는
  `Disallow: /`로 **전체 금지**. Yeti로 위장하는 것은 UA 위장(금지 사항).
- 주목: Yeti에게조차 게시판 **페이지네이션(`&page=*`)은 금지** — 네이버가 게시판 대량 크롤링을 명시적으로 원치 않음.
- **조치**: `/item/board.naver`로 **단 한 건의 요청도 보내지 않음**. 이 소스는 여기서 종료.

### 팍스넷 (paxnet.co.kr) — robots.txt 허용

```
User-agent: *
Allow: /
Disallow: /my/            ← 마이페이지만 금지 (프로필 등)
Sitemap: http://www.paxnet.co.kr/sitemap.xml
```

| 항목 | 결과 |
|---|---|
| robots.txt | 종목토론실(`/tbbs/list`, `/tbbs/view`) 허용. `/my/`만 금지(프로필 접근은 안 함) |
| 렌더링 | **정적 HTML** — 게시글 데이터(제목·시각·조회·추천·seq)가 서버 HTML에 존재. JS는 표시 포맷만 |
| 봇 차단 | 정직 UA·2.5초 간격으로 **~10요청 동안 차단·rate limit·CAPTCHA 없음** (전부 HTTP 200) |
| 이용약관 자동수집 조항 | **미확인 — 확인 필요.** robots.txt는 허용하나 ToS의 자동수집 관련 조항은 별도 검토 요망 |

## 3. 데이터 구조 (팍스넷)

- **종목토론실 URL**: `https://www.paxnet.co.kr/tbbs/list?id={종목코드}&tbbsType=L` (id = 6자리 종목코드 그대로)
- **게시글 URL**: `https://www.paxnet.co.kr/tbbs/view?id={종목코드}&seq={글번호}`
- **목록 페이지 필드** (모두 정적 HTML, 행당):
  - 글 번호, `data-seq`(글 고유 seq)
  - 제목 (`.title .tit a`)
  - 작성자 **닉네임 + 프로필 ID** (마크업에 노출됨 — *존재만 기록, 수집 안 함*)
  - 조회수(`조회`), 추천수(`추천`)
  - **작성 시각** (`data-date-format`, 초 단위)
  - 댓글 수 / 댓글 로드 함수(`viewCommentList(code, seq)`)
- **본문**: 목록엔 제목만. 본문은 `bbsWrtView(seq)`(별도 요청/AJAX)로 열어야 함 → 본문 수집은 종목당·글당 추가 요청 필요.
- **댓글**: 게시글과 분리(`viewCommentList`) — 별도 요청.
- **페이지네이션**: `goPage(N)` + hidden `<input name="page">`. **GET `&page=N` 으로 정상 동작 확인**
  (page1/2 seq·타임스탬프 상이). 페이지당 **30글**.

## 4. 데이터 가용량 (팍스넷)

| 종목 | 관측 | 추정 밀도 |
|---|---|---|
| 삼성전자(005930) | page1: 7/13→7/10, page300: ~2025-08, page1500: **2024-07** | **~30–60 글/거래일** (시기별 편차, 2024년이 더 활발) |
| 대한항공(003490) | page1 한 페이지(30글)가 **2025-07 ~ 2026-06 (~11개월)** 커버, 최신 글도 한 달 이상 지남 | **~2–3 글/월 (사실상 정지)** |

- **아카이브 깊이**: 삼성 page1500이 2024-07 → **2년+ 과거까지 초 단위 타임스탬프로 보존** 확인. 더 깊은 페이지도
  30글 정상 반환(추가 깊이 존재 시사).
- **밀도 편중이 핵심 제약**: 게시글이 소수 인기주에 집중. 대형주 삼성은 충분하나, **대한항공급 중형주조차 거의 비어
  있음.** KOSPI200 이벤트 스터디 표본 다수에서 이벤트 창(±수일) 내 게시글이 0건일 가능성 높음.
- **삭제 글**: 글 번호(NO)와 seq의 연속성으로 삭제 흔적 추정은 가능하나 이번 정찰에선 정밀 확인 안 함.

## 5. 데이터 품질 (제한적 샘플 — 의도적으로 최소 열람)

- 관측된 제목 예: `"● 속보) 삼성전자: 2분기 서프라이즈 실적발표: 매수 적기"` — **제목에 감성 신호(매수 적기, 서프라이즈)가
  실리나, 홍보성 노이즈 마커(●, 속보)도 동반.** 도배·리딩방 홍보 비율은 소표본이라 단정 불가(추가 표본 필요).
- **제목만으로 감성 판별**: 부분적으로 가능해 보이나(명시적 매수/매도 문구 빈번), 반어·맥락 의존 표현은 본문 필요.
  → 자사주 프로젝트의 뉴스 감성 룰을 재사용 가능하되, 게시판은 은어·비속어가 많아 사전 확장 필요 예상.
- **텍스트 길이**: 목록은 제목만이라 길이 분포는 본문 열람 필요 — 이번 정찰 범위 밖(원문 대량 열람 자제).

## 6. 이 소스(팍스넷)로 갈 때의 제약사항

1. **표본 편중**: 유동성 상위 종목만 유의미한 게시글 밀도. 중·소형주는 이벤트 창 내 0~수 건 → 횡단 심리 변수의
   결측·희소성 심각. 연구를 "대형주 부분표본"으로 한정하거나, 밀도 임계값 이상 종목만 포함하는 설계 필요.
2. **본문·댓글은 글당 추가 요청**: 제목 기반이면 목록만으로 충분하나, 본문/댓글 심리까지 보려면 요청 수가
   글 수만큼 증가 → 2.5초 간격 준수 시 수집 시간 대폭 증가.
3. **ToS 미확인**: robots.txt는 허용하나 이용약관의 자동수집 조항은 별도 확인 필요(수집 착수 전 필수).
4. **작성자 식별정보 노출**: 닉네임·프로필 ID가 마크업에 있음 — 연구 윤리상 **수집 제외 원칙 유지** 필요
   (이번 정찰에서도 미수집).
5. **대표성 한계**: 팍스넷은 네이버 금융보다 이용자층이 작고 특정 성향 편중 가능 → "개인투자자 심리" 일반화에
   외적 타당성 주의. 결론에 소스 편향 명시 필요.

## 7. 막힌 지점과 대안

- **막힌 지점**: 네이버 금융(개인투자자 게시판의 사실상 표준)은 robots.txt로 **원천 차단**. 우회는 금지 사항이므로
  네이버는 후보에서 제외.
- **현실적 대안 = 팍스넷** (본 리포트) — 시각 해상도는 오히려 네이버보다 나을 수 있음(초 단위 노출). 밀도 편중이
  가장 큰 약점.
- **추가 검토 여지**(미정찰): DART 공시 자체엔 감성이 없고, 뉴스는 자사주 프로젝트에서 이미 사용(네이버 뉴스 검색은
  날짜필터 크롤링이 되었음 — 게시판과 별개 경로). 커뮤니티 '날것' 심리가 꼭 필요하면 팍스넷이 현재 유일한 합법 접근원.

  전부 정직 UA(`quant-lab-research-recon/1.0`), 2.5초+ 간격, HTTP 200(팍스넷)/차단 없음.
- 네이버 종목토론방(`/item/board.naver`) 요청: **0건** (robots.txt 준수).
- 원문 텍스트 디스크 저장: **없음** (전부 stdout 휘발). `.gitignore`에 `data/raw/` 추가.
- 작성자 닉네임·ID: **미수집** (존재만 기록).
