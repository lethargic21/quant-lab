# 스팸 라벨링 가이드 (2단계 — 사람 라벨링)

> 대상 파일: `labels/spam_labels_300.csv` (무작위 300건, seed=42, **규칙 플래그 비노출 블라인드**).
> `label` 컬럼에 `spam` 또는 `ham`(정상)을 기입. 애매하면 `note`에 이유를 남기고 아래 규칙으로 판정.
> 이 라벨이 3~4단계(KcELECTRA 학습·모든 성능 수치)의 **유일한 기준**이 된다.

## 핵심 정의

**스팸 = "이 글이 해당 종목에 대한 투자자의 자발적 관심/의견이 아닌 것"**

attention 지표는 "투자자들이 이 종목에 얼마나 주목하는가"를 재려는 것이므로,
*관심의 표현이면 품질이 낮아도 ham*, *영업·도배·무관 글이면 spam*이다.

## spam 판정 (하나라도 해당하면 spam)

| 유형 | 예시 패턴 |
|---|---|
| 리딩방/영업 유도 | 오픈채팅·카톡·텔레그램 초대, 입장코드, 무료체험, VIP, 수익 인증, 종목상담 유도 |
| 광고 | 외부 서비스/사이트 홍보, 대출·보험 등 종목 무관 상업 광고 |
| 도배 | 동일·유사 문구 반복 게시 (내용 불문 — 반복 자체가 봇/도배 신호) |
| 종목 무관 | 정치 선동, 타 종목 홍보 등 해당 게시판 주제와 무관한 글 |

## ham 판정 (스팸이 아님 — 품질과 무관)

| 유형 | 판정 근거 |
|---|---|
| 단순 응원/한탄 ("가즈아", "지겹다") | **ham** — 근거 없어도 자발적 관심의 표현 |
| 근거 없는 목표가 외침 ("내일 10만") | **ham** — 틀린 의견도 의견 |
| 욕설·비난 (경영진/타 투자자 대상) | **ham** — 감정 표현도 관심 (단, 도배로 반복되면 spam) |
| 뉴스 링크 공유 + 짧은 코멘트 | **ham** — 링크가 있어도 해당 종목 정보 공유 목적이면 정상 |
| 질문 ("지금 들어가도 되나요?") | **ham** |

## 경계 사례 처리 규칙 (일관성이 정확도보다 중요)

1. **링크 판정**: 링크 자체가 아니라 *목적*으로 — 뉴스/공시 공유=ham, 리딩방·카페·프로필 유도=spam.
2. **반복 게시**: 같은 사람 글인지 알 수 없으므로(작성자 미수집), 샘플 안에서 유사 제목을 이미 봤다면
   `note`에 "반복 의심"이라 적고 **spam**. 처음 보는 제목이면 내용만으로 판정.
3. **"속보)" "단독)" 낚시 제목**: 내용이 해당 종목 얘기면 ham (저품질일 뿐), 영업 유도면 spam.
4. **판단 불가 (제목만으로 알 수 없음)**: `label`에 `unsure` 기입 — 평가에서 제외하고 그 수를 리포트에 명시.
   (unsure 남발하면 표본이 줄어드니 가능하면 판정)
5. **홍보 마커(● 등)가 있지만 내용은 평범**: 내용 기준으로 판정 — 마커는 규칙 베이스라인의 피처일 뿐,
   사람 라벨은 마커에 끌려가지 말 것 (블라인드 취지).

## 왜 이렇게 정의했나 (연구 설계와의 연결)

- attention 지표의 분자는 "자발적 관심 글 수". 리딩방 홍보 10개가 낀 날을 "관심 폭발"로 읽으면
  지표가 오염되므로 **영업성·도배성**만 걸러낸다.
- 품질 필터(저품질 의견 제거)를 시도하지 않는 이유: "저품질"은 주관적이고, 개인투자자 심리 연구에서
  거친 표현이야말로 신호일 수 있음.

## 절차

1. `labels/spam_labels_300.csv`를 엑셀/에디터로 열기 (utf-8-sig, 엑셀 바로 열림)
2. `title`(+`posted_at`)만 보고 `label`에 `spam`/`ham`/`unsure` 기입
3. 경계 사례는 `note`에 한 줄 근거
4. 저장 후 알려주면 → 3단계(KcELECTRA) 진행: 층화 분할(train/holdout), 규칙 약라벨 + 사람 라벨 결합 학습,
   **홀드아웃(사람 라벨)에서만 성능 보고**
